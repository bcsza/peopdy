# OpenLink Navigator

OpenLink Navigator 是一个面向开发人员与高级用户的轻量级外链资源导航与聚合平台，旨在以目录化、可检索的方式整理和呈现互联网上高价值的外部资源链接。项目定位为技术向的资源索引中间件，不存储任何第三方内容，仅提供结构化引用与描述性元数据，适用于个人书签增强、团队知识库外链治理、以及自动化链路巡检等场景。

目标用户包括：需要系统化管理大量外部链接的运维工程师、技术文档维护者、信息安全审计人员，以及希望快速构建可读性外链目录的开源项目维护者。通过统一的条目模型、标签体系与状态标记，OpenLink Navigator 帮助用户从杂乱的收藏夹或散落文档中抽离出结构化外链资产，并支持后续扩展为自动化健康检查或变更通知系统。

## 功能概览

- **结构化外链目录**：支持按类别、来源、协议、状态等多维度对链接进行编目，每条记录包含标题、描述、标签、添加时间与最后校验时间。

- **批量导入与去重**：提供从纯文本、CSV 及常见书签导出格式（HTML/Netscape 格式）批量导入链接的能力，并内置基于 URL 标准化的去重引擎。

- **链接可用性探测**：集成异步 HTTP/HTTPS 探活模块，可配置超时与重试策略，定期检查每条外链的可达性并记录状态变化历史。

- **标签与全文检索**：基于倒排索引的轻量级检索功能，支持按标签、域名、描述关键词进行快速筛选，适用于大规模外链库的日常查询。

- **只读 API 接口**：提供 RESTful 风格的只读 API，支持按类别、标签或最近更新获取链接列表，便于与其他监控或展示系统集成。

- **原始数据保留与导出**：所有用户导入的原始外链数据均保留原始字段（包括原始 URL 字符串、导入来源、原始描述），并支持以 JSON/YAML 格式完整导出。

## 应用场景

- **技术文档外链治理**：技术团队在维护项目文档或 Wiki 时，常引用大量外部依赖、参考文档或工具站。OpenLink Navigator 可作为统一的外链登记与校验中心，周期性检查引用的有效性，避免文档中出现死链。

- **安全研究信息聚合**：安全研究人员需跟踪多个威胁情报源、CVE 公告、漏洞利用站点。通过本项目可建立私有化外链看板，按威胁等级、数据源类型分类，并快速筛选近期新增或变更的链接。

- **个人知识库外链增强**：个人笔记或数字花园中若包含大量外部引用，可使用本工具生成独立的外链索引页，便于离线阅读时快速定位源站，同时降低笔记正文中的链接噪音。

- **运维监控通知渠道管理**：运维团队需维护告警通知集成的 Webhook 地址、面板入口、日志检索入口等。OpenLink Navigator 可作为这些关键链接的台账系统，记录变更历史并支持导出为 Prometheus 静态目标配置。

## 快速开始

以下步骤适用于 Linux / macOS / WSL 环境，确保系统已安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆仓库
git clone https://github.com/openlink-navigator/navigator-core.git
cd navigator-core

# 安装依赖
npm install

# 以开发模式运行（默认监听 3000 端口）
npm run dev
```

访问 http://127.0.0.1:3000 可查看默认示例目录。生产环境部署请参考 docs/deployment.md。

## 常见问题

**问：导入大量链接时页面卡顿或探活任务堆积怎么办？**

答：导入操作与探活任务共用事件循环，建议在导入时通过命令行工具（npm run cli import -- --batch-size=50）分批提交，并调整 scheduler 中的并发数配置（probe.concurrency）。若长期存在大规模链接库（超过 5000 条），推荐启用 Redis 作为任务队列后端，将探活任务异步分发至独立 Worker 进程。

**问：部分外链返回超时或证书错误，是否会影响整体探活状态？**

答：探活引擎对每个链接独立记录状态，超时或 TLS 错误仅标记该链接为 unreachable，并记录错误详情至 error_log 字段，不会阻塞后续链接的探测。您可在配置中通过 probe.ignore-cert-errors 控制是否忽略证书校验，默认关闭。

**问：如何迁移 SQLite 数据到 PostgreSQL？**

答：项目提供 db/migrate-to-pg.js 脚本，该脚本会自动读取 SQLite 源文件与 PostgreSQL 目标连接串，执行表结构迁移与数据复制。建议在迁移前使用 export 命令备份原始数据，迁移完成后需修改 config/default.yaml 中的 db.dialect 为 postgres 并重启服务。
