# LinkForge

LinkForge 是一个面向开发者、技术内容创作者与数字研究人员的轻量级外链资源聚合与管理平台。该项目旨在解决个人与团队在项目开发、内容调研与知识管理过程中面临的链接分散、检索低效、上下文丢失等问题，通过结构化的目录分类、标签系统与快照机制，将零散的 URL 转化为可复用、可共享、可追溯的结构化知识资产。LinkForge 适用于需要长期维护大量外链资源的个人知识库、技术博客的参考文献管理、开源项目的相关资源导航，以及研究机构的外部数据源登记场景。

## 功能概览

**多级目录分类**：支持无限层级的自定义目录树，用户可根据主题、领域或使用场景灵活组织链接资源，实现直观的浏览与导航。

**全文检索与标签过滤**：内置基于倒排索引的轻量级检索引擎，支持对链接标题、描述、标签与备注内容的快速检索，同时支持多标签组合筛选。

**链接快照与自动归档**：在添加链接时自动抓取目标页面的标题与元描述，并生成时间戳快照记录，防止目标页面内容变更后丢失原始引用上下文。

**批量导入与导出**：支持从浏览器书签 HTML 文件、CSV 表格及纯文本列表批量导入链接，同时支持将当前目录下的链接列表导出为 Markdown、JSON 或 CSV 格式。

**链接健康检查**：定时对已存储的链接进行可达性探测与状态码检查，自动标记失效或响应超时的链接，并提供异常报告。

**访问统计与热度分析**：记录每个链接的点击次数、最后访问时间与来源渠道，生成简单的访问趋势图表，辅助用户识别高频使用的核心资源。

**团队协作与权限管理**：支持多用户空间隔离，允许创建共享目录并设定只读或可编辑权限，便于团队内部共同维护外部资源清单。

**开放 API 接口**：提供基于 RESTful 风格的 JSON API，支持第三方工具对链接数据进行增删改查操作，便于集成至现有工作流。

## 应用场景

技术文档撰写与维护：技术作者在撰写教程或博客时，需要引用大量外部文档、代码仓库、规范标准等链接。LinkForge 可以为每篇文章建立独立的资源目录，自动保存快照，确保在文章发布后即使外部链接变更，作者依然能够追溯到引用时的原始内容。

开源项目依赖与参考管理：开源项目维护者通常需要记录上下游依赖、相关工具链、社区论坛、示例项目等外部资源。通过 LinkForge 建立项目专属的外链清单，并利用健康检查功能定期验证依赖链接的有效性，可显著降低项目文档中的链接腐坏问题。

学术研究与文献调研：研究人员在文献调研阶段会收集大量论文页面、数据集地址、工具官网、机构主页等链接。LinkForge 的多级目录与标签系统可以按研究主题、文献类型、重要程度等维度进行分类，批量导出功能则可快速生成参考文献列表或数据来源声明。

个人知识库外链整合：使用 Obsidian、Notion 或本地 Markdown 文件构建个人知识库的用户，往往面临外链散落、难以统一管理的问题。LinkForge 可作为知识库的外部链接补充层，通过 API 或导出功能与主流笔记软件协同工作，实现本地笔记与云端外链的分离管理。

## 快速开始

以下步骤将指导您在本地环境中快速启动 LinkForge 服务。

```bash
# 克隆代码仓库
git clone https://github.com/linkforge/linkforge.git
cd linkforge

# 安装项目依赖（使用 Python 虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化数据库并启动开发服务器
python manage.py migrate
python manage.py createsuperuser  # 按提示创建管理员账号
python manage.py runserver
```

服务启动后，在浏览器中访问 http://127.0.0.1:8000 即可进入 LinkForge 主界面。首次使用请使用上方创建的管理员账号登录，并开始创建您的第一个链接目录。

## 常见问题

启动服务后访问页面出现 500 错误，日志显示数据库连接失败。

请检查 PostgreSQL 服务是否正常运行，并确认 settings.py 中的 DATABASES 配置中的用户名、密码、主机地址与端口号与当前环境一致。开发环境下默认使用 SQLite，若未安装 PostgreSQL 可临时切换至 SQLite 继续使用。

添加链接时快照抓取失败，提示超时或无法获取页面标题。

快照抓取依赖外部网络访问，请确保部署环境能够访问目标网站。对于国内用户，部分境外站点可能因网络原因超时，可在系统设置中调整抓取超时时间或关闭自动快照功能改为手动编辑。同时请确认目标站点是否屏蔽了自动化请求的 User-Agent。

健康检查任务执行后大量链接被标记为失效，但手动访问却是正常的。

健康检查采用 HTTP 状态码与响应超时双重判定机制。若目标站点启用了反爬机制或要求特定 Header，可能导致检查失败。您可以在健康检查配置中调整请求头信息，或将信任域名加入白名单跳过检查。建议同时检查目标站点的 SSL 证书是否有效，证书过期也会导致检查失败。
