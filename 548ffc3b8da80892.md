# Tangram Navigator

Tangram Navigator 是一个面向中文互联网用户的轻量级技术资源导航与信息聚合项目。本项目不提供任何实质内容存储或分发服务，仅作为公开可用网络资源的结构化索引与访问入口，便于用户快速定位特定类型的在线工具、信息平台与社区站点。项目定位为个人维护的信息整理集合，适合希望提升信息检索效率、发现新类别网络资源的终端用户与技术爱好者使用。

## 功能概览

- **分类资源索引**：按站点类型与内容主题对收录链接进行分层整理，提供清晰的浏览路径。
- **静态导航页面**：基于 Markdown 生成的轻量静态 HTML，无需后端服务即可部署与访问。
- **外部链接直出**：所有收录 URL 保持原始格式完整输出，不做自动补全或协议改写，确保地址精确性。
- **批次化管理**：以批次为单位组织新增资源，本批次为第 29/48 批，便于追溯更新记录。
- **无侵入性集成**：项目本身不加载任何外部第三方脚本或追踪代码，保持纯文档特性。
- **社区共建入口**：提供标准贡献流程，允许用户通过 Pull Request 推荐新站点或更新失效链接。

## 应用场景

- 个人浏览器起始页替代方案：用户可将 Tangram Navigator 部署为本地或内网起始页，集中访问高频使用的工具站点。
- 新站点发现与探索：对于关注特定领域（如影视信息、技术社区）的用户，可通过索引快速发现未知但活跃的网络资源。
- 团队内部知识共享：技术团队或兴趣小组可使用本项目作为基础模板，定制内部常用的开发文档、设计资源与协作平台链接集合。

## 快速开始

以下指令适用于 Linux/macOS 及 Windows WSL 环境，需提前安装 Git 与 Node.js（建议 LTS 版本）。

```bash
# 克隆仓库
git clone https://github.com/tangram-navigator/tangram-navigator.git
cd tangram-navigator

# 安装依赖（使用 npm）
npm install

# 生成静态站点（输出至 dist 目录）
npm run build
```

## 贡献指南

1. 复刻本仓库至个人账号，并克隆到本地开发环境。
2. 在 `src/data/batches/` 目录下新增或修改对应批次的 JSON 文件，遵循既定字段结构（包含 title、url、category、description 等）。
3. 本地执行 `npm run validate` 校验所有链接格式与可用性，确保新增 URL 不包含协议前缀不一致或非法字符。
4. 提交变更并推送至个人复刻仓库，随后通过 GitHub 界面发起 Pull Request 至主仓库的 `main` 分支。
5. 等待项目维护者审核，若链接符合内容规范且无重复，将合并进入下一批次发布。

## 常见问题

Q: 为什么项目不直接提供搜索或代理功能？
A: 本项目的定位仅为静态导航索引，不涉及任何后端服务、爬虫或代理转发。所有链接均需用户自行访问，项目方无法保证第三方站点的可用性与内容合法性。

Q: 如何报告失效链接或不当内容？
A: 请通过 GitHub Issues 提交具体 URL 及失效描述（如返回 404、连接超时等）。维护者会在定期检查中核实并移除或替换失效条目。

Q: 每批次的更新频率是多久？
A: 本项目按不定期批次更新，每批次收录 3-6 个新链接。批次编号连续，但无固定时间表，具体更新视资源收集情况而定。
