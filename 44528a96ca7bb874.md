# HXMedia Navigator

HXMedia Navigator 是一个面向数字媒体从业者、技术研究人员与内容聚合平台运营方的轻量级导航与资源索引系统。该项目定位于解决媒体类站点分散、域名变更频繁、可用性监测困难等实际问题，通过结构化的数据组织与清晰的资源分类，帮助用户快速定位可用的媒体服务入口。项目本身不存储或分发任何受版权保护的内容，仅提供公开可访问的 URL 索引与基础可用性检测辅助功能。

目标用户包括：需要维护媒体服务书签集合的个人用户、从事网络资源可用性研究的技术人员、以及需要批量管理外部媒体链接的小型团队。项目通过静态页面生成与轻量级后端检测脚本相结合的方式，提供稳定、可扩展的资源导航能力。

## 功能概览

- **资源分类索引**：按站点类型、服务区域、内容主题等多维度对收录的 URL 进行分层归类，支持快速筛选与定位。

- **可用性状态检测**：内置简单的 HTTP 状态码检测模块，可定时探测各资源站点的响应情况，并在前端面板中以视觉化标记呈现。

- **批量导入与导出**：支持 CSV 与 JSON 格式的资源列表批量导入，同时可将当前索引导出为标准格式，便于迁移与备份。

- **自定义标签体系**：允许用户为每个 URL 添加自定义标签，实现个性化分组，满足不同使用场景下的分类需求。

- **搜索与过滤**：提供基于站点名称、标签、描述字段的关键词搜索，以及按可用状态、分类标签的过滤功能。

- **响应式布局**：前端界面适配桌面端与移动端浏览器，确保在不同屏幕尺寸下均能获得良好的浏览与操作体验。

## 应用场景

- **媒体从业者日常资源导航**：内容采编人员可通过本系统快速访问多个媒体服务入口，避免记忆大量域名或依赖浏览器散乱的书签文件夹。

- **可用性监控辅助工具**：运维人员可利用内置的检测模块定期核查资源站点状态，及时发现不可用的服务入口并做出调整。

- **技术研究参考数据源**：研究人员可将本系统作为研究域名活跃度、站点生命周期等课题的基础数据索引，便于构建实验样本集。

- **团队内部资源共享**：小型团队可将本系统部署为内部共享的资源清单，统一维护团队所需的各类外部服务链接，减少信息孤岛。

## 快速开始

以下步骤适用于开发环境部署与本地运行测试。生产环境部署请参考文档导航章节中的部署指南。

```bash
# 克隆代码仓库
git clone https://github.com/example/hxmedia-navigator.git
cd hxmedia-navigator

# 安装项目依赖
pip install -r requirements.txt

# 初始化本地配置文件
cp .env.example .env

# 运行可用性检测脚本（首次运行）
python scripts/check_availability.py --input data/urls.json --output data/status.json

# 启动本地开发服务器
python app.py --host 127.0.0.1 --port 8080
```

启动成功后，在浏览器中访问 `http://127.0.0.1:8080` 即可查看导航界面。

## 常见问题

**Q：可用性检测的判定标准是什么？**

检测脚本会向目标 URL 发送 HTTP HEAD 请求，超时时间为 5 秒。判定为可用的条件为：返回状态码在 200 至 399 范围内，且响应头中 Content-Type 不为空。重定向（301、302）状态码被视为可用，并记录最终重定向目标。

**Q：如何更新本地缓存的资源列表？**

您可以通过两种方式更新资源列表：一是直接编辑 `data/urls.json` 文件，按既定格式添加或删除条目；二是使用 `scripts/import_csv.py` 工具从 CSV 文件批量导入。更新后需重新运行检测脚本以刷新状态缓存。

**Q：项目是否支持多用户登录与权限管理？**

当前版本为单机部署模式，不包含用户认证系统。若需多用户访问，建议通过反向代理配置基础认证，或部署在受信任的内网环境中。多用户版本已在后续规划中。
