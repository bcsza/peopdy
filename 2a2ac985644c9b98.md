# 环球视野导航系统

Global Horizon Navigation System (GHNS) 是一个面向中文互联网用户的高质量垂直领域资源聚合与导航平台。本项目并非传统意义上的搜索引擎或通用网址导航，而是专注于影视文娱、特色电商、垂直社群与个性化内容消费等细分赛道，通过人工筛选与机器辅助验证相结合的方式，为特定兴趣群体提供稳定、可发现、高相关性的外部优质站点入口。

本项目解决的核心痛点在于：优质细分领域资源分散、域名记忆成本高、站点存活状态不稳定、以及跨平台跳转过程中的信息损耗问题。GHNS 不生产内容，而是做内容管道的最优路径映射，帮助用户以最低认知负担抵达目标服务。

## 功能概览

- **智能分类索引** 按内容形态、服务类型与受众画像将收录站点划分为影视流媒体、电商导购、兴趣社区与工具服务等大类，支持多维度筛选与排序。

- **存活健康巡检** 对收录的外部链接进行定时可用性探测，自动标记疑似失效节点，并在前端界面以状态指示灯形式直观呈现，减少用户点击无效链接的挫败感。

- **简洁直达体验** 所有外链采用原始直出模式，不附加任何跟踪参数、中间跳转页或反爬验证码，确保从点击到内容呈现的路径最短化。

- **标签与全文检索** 为每个收录站点赋予多级标签体系（语言、地区、收费模式、移动端适配程度），支持关键词模糊匹配，帮助用户在不确定确切域名时快速定位目标。

- **用户提交与社区修正** 开放外部站点提交入口，允许注册用户推荐新资源或对已有条目的分类、描述、可用性进行纠错，经审核后合并至主库。

- **版本化快照备注** 对重要站点记录其历史域名变更轨迹与服务范围演变说明，帮助老用户理解迁移背景，降低因品牌更名或业务转型带来的认知断裂。

- **无广告干扰承诺** 本导航系统自身不投放任何商业广告位，不进行竞价排名，所有收录排序依据为客观质量与社区口碑，非商业化合作结果。

- **明暗主题与阅读模式** 提供日间/夜间主题切换，并针对长文本描述区域优化排版与对比度，提升长时间浏览的视觉舒适度。

## 应用场景

- **影视爱好者快速追剧入口** 用户通过影视分类子页，可一键直达多个中小型影视资源站，获取不同压制组、字幕组或流媒体源的内容，避免因单个站点维护或地域限制导致无法访问，极大提高资源获取效率。

- **特色电商选品与比价辅助** 电商导购类目收录了若干垂直品类独立站，用户可在不同店铺间横向对比商品细节、价格区间与用户评价，尤其适合对大众主流平台同质化商品感到厌倦、希望发掘小众设计或手工制品的购物群体。

- **兴趣社群活动信息聚合** 部分收录站点承载了特定亚文化圈层的活动公告、作品展示与线下聚会信息。本导航作为统一入口，帮助新成员快速融入社群生态，减少因信息孤岛造成的参与门槛。

- **临时设备或无搜索环境下的应急访问** 当用户处于受限网络环境、临时使用他人设备或不方便启用搜索引擎时，可直接记忆本导航主域名，作为跳板站快速触达日常高频使用的服务，避免重复输入长串复杂域名。

## 快速开始

以下步骤适用于首次部署本导航系统的开发环境或本地预览实例。

```bash
# 克隆主仓库
git clone https://github.com/ghns/global-horizon-nav.git

# 进入项目目录并安装依赖（使用 npm）
cd global-horizon-nav
npm install

# 启动开发服务器（默认监听 3000 端口）
npm run dev
```

生产环境部署请参考 `docs/deployment.md` 文档，支持 Docker 容器化一键启动与 Nginx 反向代理配置模板。

## 常见问题

**Q: 收录的站点无法访问或者显示状态异常，应该怎么办？**

A: 您可以直接在 Issues 中提交站点不可用报告，或通过用户提交页面标记该站点为失效。系统每日凌晨会执行自动健康检查，但人工反馈能显著加速异常节点的剔除或替换流程。若站点仅临时离线但域名本身有效，我们会在备注栏添加“近期不稳定”提示，避免直接移除以保留历史记录。

**Q: 为什么某些站点需要反复点击才能打开，或者被浏览器拦截？**

A: 本导航仅做纯文本链接陈列，不干涉外部站点的服务内容与网络传输层。部分站点可能使用非标准端口、自签名证书或处于特定网络环境，导致浏览器发出安全警告。建议用户在充分了解风险的前提下手动确认继续访问。本项目不对第三方站点的内容合法性、安全性及可用性承担任何担保责任。

**Q: 我可以将本导航系统部署到自己的服务器上，并修改收录的站点列表吗？**

A: 可以。本项目基于 MIT 许可证发行，您完全有权复制、修改、分发及用于商业或非商业目的。但需注意，您自行维护的实例所产生的所有内容责任与运营风险由您自行承担，建议在显著位置标明该实例与原始项目无隶属关系。
