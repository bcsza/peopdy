# PaperLink Navigator

PaperLink Navigator 是一个面向科研工作者与工程技术人员的开源学术资源导航站，专注于聚合高质量、高可用性的外文文献获取渠道与学术工具镜像站。项目本身不存储任何版权内容，仅提供公开可访问的 URL 索引与可用性监测服务，帮助用户快速定位稳定、低延迟的学术资源入口。

项目定位为技术化的外链汇总工具，而非内容聚合平台。目标用户包括高校研究生、实验室研究员、跨国协作团队以及需要频繁访问海外学术数据库的开发者。通过结构化分类、可用性标注与简洁的静态页面呈现，PaperLink Navigator 致力于减少用户在混乱书签与失效链接间反复试错的时间成本。

## 功能概览

- **站点可用性实时探测**：每 30 分钟对收录的 URL 执行 HTTPS 握手与响应码检查，自动标记异常节点。
- **按来源与用途分类索引**：将链接划分为论文检索、预印本、镜像站、工具辅助等类别，支持快速筛选。
- **简洁只读的静态展示页**：生成纯 HTML + CSS 的导航页面，无 JavaScript 依赖，兼容低版本浏览器。
- **响应式布局与快速跳转**：适配桌面与移动设备，所有链接均以原始地址直出，无重定向或跟踪参数。
- **开源监测脚本**：提供 Python 探测脚本，用户可自行部署定时任务，监控自定义 URL 列表。
- **可用性历史趋势图**：基于 SQLite 记录每次探测结果，生成近 7 天或 30 天的可用率折线图。
- **批量导入与导出**：支持以 YAML 格式批量新增链接条目，并可导出为 CSV 或 JSON 用于二次处理。

## 应用场景

- **实验室内部知识库**：课题组可将本导航部署在内网服务器，作为组内共享的文献入口页，避免成员反复询问常用数据库地址。
- **跨国协作中的备用通道**：当主要学术站点在部分地区访问不稳定时，通过导航中的镜像站或替代域名快速切换，保障论文下载与同步工作不中断。
- **个人书签的定期校验**：研究人员可借助项目的监测脚本，每周自动检查个人收藏的 30+ 个学术链接，自动生成失效报告，便于及时更新书签。
- **开源镜像站维护参考**：镜像站运维人员可参考本项目的 URL 列表，补充或对比自身的上游源配置，提升镜像覆盖度与冗余性。

## 快速开始

以下命令适用于 Linux / macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash。

```bash
# 克隆仓库
git clone https://github.com/paperlink-navigator/navigator.git
cd navigator

# 安装依赖（推荐使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 执行完整扫描并生成静态页面
python scanner.py --urls config/urls.yaml --output dist/index.html
```

完成上述三步后，使用浏览器打开 `dist/index.html` 即可查看导航页。若需开启定时监测，可配置 crontab 定时执行 `python scanner.py --update`。

## 贡献指南

1. 在 GitHub 上 fork 本仓库，并克隆到本地开发环境。请确保使用 Python 3.9+ 且已安装所有开发依赖（`pip install -r requirements-dev.txt`）。
2. 新增或修改 URL 条目时，请编辑 `config/urls.yaml`，并严格遵守既有 Schema：每个条目须包含 `url`、`category`、`description` 三个字段。提交前请运行 `python scanner.py --validate` 进行格式校验。
3. 若需添加新的探测策略（如支持 TCP 端口探测或 DNS 解析时长记录），请在 `core/probe.py` 中扩展 `ProbeStrategy` 类，并在 `tests/test_probe.py` 中补充对应的测试用例。
4. 文档更新请同步修改 `docs/` 下对应文件，中文表述务必清晰、无歧义。涉及命令行示例的部分，请确保所有路径与参数与当前代码一致。
5. 提交 Pull Request 前，请执行完整测试套件（`pytest tests/`）并确保所有用例通过。PR 描述中请明确说明改动影响范围及测试覆盖情况。

## 常见问题

**问：探测结果出现大量超时，但浏览器中可以直接访问，如何解决？**

答：这通常是因为服务器对 `requests` 库的默认 User-Agent 或 TLS 指纹有限制。请在 `config/settings.yaml` 中调整 `user_agent` 字段为常见浏览器 UA（如 Chrome 或 Firefox 最新版），并检查 `timeout` 是否设置过短（建议不低于 10 秒）。若仍失败，可将该域名加入 `whitelist.yaml` 并启用 `disable_ssl_verify: true`，但请注意这会降低安全性。

**问：如何只对某一个分类的链接进行探测，而不扫描全部？**

答：执行扫描时添加 `--category` 参数，例如 `python scanner.py --category mirror` 将仅扫描类别为 `mirror` 的条目。分类名称需与 `urls.yaml` 中定义的完全一致。若需同时扫描多个分类，可使用 `--category mirror,preprint` 用逗号分隔。

**问：生成的 HTML 导航页中的链接为何是纯文本而非可点击的超链接？**

答：这是项目设计上的刻意选择，目的是完整保留原始 URL 字符串便于复制与核对，同时避免浏览器对链接的自动预解析干扰页面加载。所有 URL 在页面中以等宽字体展示，用户可直接选中并粘贴至地址栏访问。若需生成可点击链接，请使用 `--clickable` 开关，但请注意这会增加页面大小并可能触发某些安全策略。
