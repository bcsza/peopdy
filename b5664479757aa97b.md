# OpenWatchHub

OpenWatchHub 是一个面向全球互联网视频爱好者的开源影视资源导航与聚合检索项目。项目本身不存储、不传播、不分发任何视频文件，仅提供公开可访问的第三方影视站点链接与基础检索能力，帮助用户快速定位可用的在线观影入口。项目定位为技术向的资源外链汇总工具，主要服务于对在线影视访问稳定性、站点可用性及入口多样性有持续需求的终端用户与研究人员。

OpenWatchHub 通过自动化可用性检测与人工审核相结合的方式，维护一份高质量、低冗余的影视站点名录。项目不设会员体系、无广告植入、不追踪用户行为，所有功能均以静态页面与纯前端脚本形式交付，适合部署于 GitHub Pages、Cloudflare Pages 等无服务器环境。项目同时提供结构化站点元数据输出（JSON / YAML），便于第三方工具集成与二次开发。

## 功能概览

- **多源影视站点导航**：收录涵盖主流影视、纪录片、动漫、综艺等类型的第三方在线播放站点，按地区、语言、内容类型分类展示。

- **站点可用性实时检测**：前端定时发起轻量级 HEAD 请求检测目标站点连通性，不可用站点自动置灰并记录离线时间。

- **关键词快速检索**：支持按站点名称、域名关键词、内容标签（如 "4K"、"美剧"、"日漫"）进行模糊匹配检索，结果即时刷新。

- **站点详情信息展示**：每个条目包含站点名称、域名、内容语言、主要分类、备案信息（如有）、更新频率及用户评分摘要。

- **自定义收藏与分组**：用户可将常用站点加入本地收藏夹（localStorage 持久化），并自定义分组标签，满足个性化使用习惯。

- **暗色主题与布局切换**：内置明暗两套配色方案，支持列表视图与卡片视图一键切换，适配不同使用环境与屏幕尺寸。

- **结构化数据导出**：提供站点列表的 JSON 与 YAML 格式导出接口，方便开发者导入其他工具或进行二次加工。

- **RSS 订阅更新通知**：站点新增、下线或域名变更时，通过 RSS 订阅源推送更新摘要，用户可在阅读器中订阅。

## 应用场景

- **影视爱好者日常访问入口**：用户可将 OpenWatchHub 作为浏览器首页或书签工具，快速跳转至可用的在线影视站点，避免逐个尝试失效域名。

- **网络运维与质量监测辅助**：运维人员可定期运行项目中的可用性检测脚本，生成站点连通性报表，用于评估不同区域网络环境下影视服务的可达性。

- **开源技术学习与前端实践**：开发者可 Fork 本项目学习纯前端路由、本地存储、跨域请求检测、响应式布局等实现方案，亦可用于演练 GitHub Actions 自动化构建流程。

- **内容聚合站点的数据源补充**：小型影视导航站或自媒体作者可定期导入本项目导出的站点 JSON 数据，作为自身内容库的补充来源。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/openwatchhub/openwatchhub.git

# 进入项目目录
cd openwatchhub

# 安装依赖（项目使用 pnpm 管理，也支持 npm / yarn）
pnpm install

# 启动开发服务器，默认端口 3000
pnpm run dev
```

启动后，在浏览器中访问 `http://localhost:3000` 即可查看导航页面。生产环境构建使用 `pnpm run build`，产物输出至 `dist` 目录，可部署至任意静态托管服务。

## 贡献指南

1.  Fork 本仓库至个人账户，并在本地克隆 Fork 后的副本。建议新建 feature 分支进行开发，分支命名格式为 `feature/简短描述`。

2.  新增或修改站点数据时，请编辑 `src/data/sites.json` 文件，严格遵守既定 JSON Schema（包含 name、url、category、language、tags 等字段），并确保 URL 为原始可访问地址。提交前运行 `pnpm run validate` 校验数据格式。

3.  提交代码前运行 `pnpm run lint` 与 `pnpm run test` 确保代码风格一致且现有功能未回退。所有提交信息请使用英文，遵循 Conventional Commits 规范（如 `feat: add new site entry`、`fix: correct health check timeout`）。

4.  发起 Pull Request 至主仓库的 `main` 分支，并在 PR 描述中清晰说明改动内容、测试情况以及是否影响现有功能。PR 合并前需通过所有 GitHub Actions 自动化检查。

5.  如发现站点链接失效或有新的可用站点推荐，可在 Issues 中提交 "Site Request" 或 "Broken Link" 类型问题，维护团队会定期审核处理。

## 常见问题

**问：项目是否提供视频播放或下载功能？**

答：不提供。OpenWatchHub 纯为导航工具，不托管、代理或缓存任何视频文件。所有播放与下载操作均在第三方站点完成，用户需遵守目标站点的使用条款。

**问：为什么某些站点在列表中显示不可用？**

答：项目前端会定期发送跨域请求检测站点连通性。由于浏览器跨域策略限制，部分站点可能误报为不可用，实际仍可正常访问。建议以实际点击结果为准。同时，部分站点可能因地域或网络环境差异导致访问受限。

**问：如何更新本地收藏的站点数据？**

答：收藏数据存储在浏览器 localStorage 中，不受站点列表更新影响。如需同步最新站点列表，请手动刷新页面获取最新 `sites.json`。若需重置收藏，可在浏览器开发者工具中执行 `localStorage.clear()` 或使用页面设置中的重置按钮。
