# LinkHub

LinkHub 是一个面向技术调研人员、运维工程师与网络安全从业者的外链资源导航与状态监测平台。该项目并非传统的爬虫或采集系统，而是一套侧重于资源可访问性、变更感知与分类管理的轻量级链接治理工具。LinkHub 帮助用户将分散于各处的技术文档、社区入口、内部系统地址以及第三方 API 端点统一纳入可观测的目录体系，从而降低因链接失效、域名迁移或协议变更导致的业务中断风险。项目本身不存储任何代理内容，仅提供基于 HTTP 标准库的连通性检查与响应摘要提取，适用于内网或公网场景下的链接资产盘点。

## 功能概览

- **链接目录分级管理** 支持按项目、团队或业务域创建多级目录，每个条目可附加负责人、归属部门和预期响应状态码。

- **批量连通性探测** 基于可配置的超时阈值与重试策略，对目录内所有 URL 执行周期性 GET 请求，记录状态码、响应时间和 TLS 证书剩余天数。

- **变更历史审计** 当目标 URL 返回的 Content-Type、响应体哈希或 HTTP 状态码发生变化时，自动生成差异记录，便于回溯异常根源。

- **标签化筛选与全文检索** 每个资源可打上多个自定义标签，配合标题与描述字段的模糊匹配，实现海量链接中的快速定位。

- **只读只写 API 接口** 提供 RESTful API 供第三方系统批量导入或导出链接数据，所有写操作均记录操作人 IP 与时间戳。

- **状态看板与告警规则** 内置简易仪表盘展示总体可用率、平均响应延迟和近期异常 Top 10，支持针对特定域名或状态码设置邮件告警阈值。

## 应用场景

- **技术团队内部文档库维护** 当团队拥有数十个内部 Wiki、代码仓库和 CI 系统入口时，LinkHub 可定期检查这些地址是否仍可访问，并在某个测试环境域名被回收前提前通知负责人。

- **第三方 API 供应商监控** 对于依赖多家外部服务接口的业务系统，使用 LinkHub 集中记录各 API 的基础 URL 与健康检查端点，一旦某个供应商返回 5xx 或证书即将过期，运维人员可第一时间获得结构化告警。

- **合规审计前的链接盘点** 在每年度的安全合规审查前，安全团队可通过 LinkHub 导出一份完整的对外暴露域名清单，结合探测结果快速筛选出已下线但未删除的残留 DNS 记录。

- **社区资源聚合站点的日常巡检** 若运营着一个汇集了技术博客、开源项目镜像站与工具下载源的内容聚合页，LinkHub 可用于每日自动检测外链的有效性，避免用户点击后遇到 404 或域名停放页。

## 快速开始

以下命令将在本地环境完成 LinkHub 核心服务的克隆、依赖安装与启动。请确保系统已安装 Git 与 Node.js 18.x 及以上版本。

```bash
git clone https://github.com/linkhub-io/linkhub-core.git
cd linkhub-core
npm install --production
npm run build && npm run start
```

若需以开发模式运行（包含热重载与调试日志），请将最后一条命令替换为：

```bash
npm run dev
```

服务启动后，默认监听本机 3000 端口，可通过浏览器访问 `http://127.0.0.1:3000` 查看初始配置向导。

## 常见问题

**Q：探测引擎是否会频繁触发目标网站的限流机制？**

A：LinkHub 默认使用指数退避重试策略，且每个目标 URL 的探测间隔不低于 300 秒。用户可在配置文件中调整 `probe.interval` 和 `probe.concurrentLimit` 参数，以控制整体请求频率。对于敏感的生产环境接口，建议在探测时附加 `X-LinkHub-Check: true` 自定义请求头，便于目标服务识别并放行。

**Q：数据库是否支持从 SQLite 迁移至 MySQL 或 PostgreSQL？**

A：项目核心抽象了统一的数据库适配层，目前官方仅提供 PostgreSQL 的迁移工具与测试用例。用户可在 `src/db/adapters/` 目录下自行实现其他关系型数据库的适配器，并在配置文件中修改 `db.dialect` 字段。迁移过程中请注意字段类型映射，特别是布尔值与时间戳的差异。

**Q：如何自定义告警接收渠道，例如钉钉或企业微信？**

A：`notifier` 模块支持通过 webhook 方式扩展任意第三方渠道。用户需在配置文件中定义 `notifier.webhooks` 数组，每个对象包含 `url`、`method` 和 `headers` 字段。当触发告警时，系统会以 POST 请求发送 JSON 格式的告警载荷。若需更复杂的模板逻辑，可继承 `BaseNotifier` 类并重写 `formatMessage` 方法。
