# LinkVault

LinkVault 是一个面向技术团队与个人开发者的外链资源聚合与管理平台，专注于解决分散收藏、链接失效、检索困难等常见问题。项目定位为轻量级的技术资源导航工具，通过结构化分类与集中存储，帮助用户高效管理与复用高频访问的外部链接。

LinkVault 并非传统书签管理器，而是以项目维度组织资源，支持标签系统、状态标记与简单检索，适用于需要维护大量技术文档、API 参考、在线工具与社区页面的工作场景。项目本身不存储任何第三方内容，仅提供链接结构化整理与展示能力，所有外链资源由用户自行维护与审核。

## 功能概览

- **链接分组管理**：支持按技术领域、项目阶段或使用频率自定义分组，每组可独立命名与排序。

- **标签系统**：每条链接可附加多个标签，支持按标签快速筛选，标签支持层级结构。

- **状态标记**：可标记链接为稳定、待审核、已失效或备用，便于团队协作时识别链接可用性。

- **全文检索**：基于链接标题、描述、标签与分组名称的轻量级索引，支持模糊匹配。

- **批量导入导出**：支持 CSV 与 JSON 格式的批量链接导入导出，便于迁移与备份。

- **外链可用性检测**：提供定时或手动触发的外部链接可达性检测，并记录检测时间与状态。

- **访问统计**：记录每条链接在近 30 天内的点击次数，辅助判断资源热度。

- **响应式展示**：前端界面适配桌面与移动设备，支持浅色与深色主题切换。

## 应用场景

- **团队技术文档维护**：开发团队可将项目相关的设计文档、API 规范、部署手册、监控面板等链接集中收录于 LinkVault，新人入职时可快速获取全部必要资源入口。

- **个人学习资源整理**：在学习新技术栈时，用户可将官方教程、博客文章、视频课程、代码示例、社区讨论帖等分散在不同浏览器的收藏夹中的链接统一迁移至 LinkVault，并通过标签区分学习阶段与优先级。

- **运维故障处理备忘**：运维人员可将常见故障的排查手册、日志查询入口、监控告警地址、各云服务商控制台等链接分类存储，在紧急故障时通过关键字快速定位对应工具。

- **开源项目外部依赖索引**：开源项目维护者可在仓库中附带 LinkVault 导出的链接集合，明确项目所依赖的第三方库文档、构建工具、持续集成服务、代码质量检测平台等外部资源地址。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git

# 进入项目目录
cd linkvault

# 安装依赖
npm install

# 启动开发服务
npm run dev
```

生产环境构建与启动：

```bash
npm run build
npm start
```

Docker 方式运行：

```bash
docker build -t linkvault .
docker run -p 3000:3000 linkvault
```

## 常见问题

**问：LinkVault 能否直接替代浏览器自带的书签管理器？**

答：LinkVault 定位于团队协作与项目维度的链接管理，而非个人浏览器的书签同步工具。它更适用于需要多人共同维护同一套资源索引的场景，或需要附加状态标记、检测可用性、统计访问频次等扩展能力的场景。个人用户也可将其作为书签的补充整理工具，但不会自动同步浏览器内置书签。

**问：外链可用性检测是否会影响目标网站的访问统计或被误判为恶意爬虫？**

答：检测采用标准的 HEAD 请求方法，仅获取响应状态码与响应时间，不会请求完整页面内容，因此不会影响目标网站的访问统计量。检测频率默认每小时仅对同一域名发起一次请求，且不并发大量检测，以降低被目标服务器误判为异常流量的风险。用户可自行调整检测间隔或禁用特定域名的检测。

**问：如果某个外部链接失效，LinkVault 是否会主动通知相关使用者？**

答：系统检测到链接状态变更（如从可达变为不可达）后，会在界面中更新状态徽标，并在下次用户访问该链接所在分组时显示提示。对于团队部署版本，支持配置 Webhook 或邮件通知，将状态变更信息发送至指定的企业微信群、Slack 频道或邮箱列表。通知接收人可在系统设置中自定义。
