# OpenLink Navigator

OpenLink Navigator 是一个面向开发者与技术研究人员的开源外链资源聚合与导航系统。该项目定位于解决技术信息分散、优质外部资源难以系统化整理与检索的问题，通过结构化的目录体系与标签化索引机制，帮助用户从海量网络节点中快速定位高价值技术站点、工具库与社区平台。系统本身不存储任何第三方内容，仅提供链接的元数据描述与分类映射，适用于个人知识管理、团队技术选型参考以及公开技术情报的采集与归档。

本项目默认以只读方式运行，所有外链数据均通过纯文本配置文件进行管理，支持静态站点生成、容器化部署以及轻量级 API 接入。目标用户包括运维工程师、全栈开发者、技术文档撰写者以及开源社区维护者。通过 OpenLink Navigator，用户可将零散的浏览器书签转化为可版本控制、可协同维护、可自动校验可用性的结构化资源清单，从而降低信息遗忘与链接失效带来的认知负担。

## 功能概览

**多级分类目录** 支持无限层级的资源分类与子分类，每个资源可归属至多个标签体系，便于从不同维度检索。

**链接可用性监控** 内置定时检测任务，可对已收录的外链发起 HTTP 头请求，自动标记响应异常或超时的条目，辅助清理失效资源。

**全文元数据检索** 基于资源标题、描述、标签与域名信息构建轻量级倒排索引，支持模糊匹配与字段限定查询，响应时间在十万级条目下低于 200 毫秒。

**静态站点导出** 提供命令行工具将配置数据渲染为纯 HTML 静态页面，无需数据库即可直接部署至 Nginx 或对象存储服务。

**RESTful 管理接口** 提供基于 JSON 格式的增删改查 API，支持批量导入与导出，便于与其他内部系统进行数据同步。

**访问统计看板** 记录每个外链的点击次数与最后访问时间，以简单柱状图形式展示热门资源排行，辅助判断资源价值。

**标签自动补全** 在编辑或新增资源时，根据已有标签库提供前缀匹配建议，减少手动输入错误，保持标签一致性。

**配置热加载** 修改资源配置文件后无需重启服务，系统通过文件系统事件触发索引重建，保证数据实时生效。

## 应用场景

个人技术书签管理与跨设备同步。开发者可将日常浏览积累的技术博客、在线工具、API 文档等链接统一录入 OpenLink Navigator，通过自托管的 Web 界面在任何终端上访问，彻底摆脱浏览器厂商的书签同步限制与隐私顾虑。

团队内部技术栈知识库建设。技术负责人可使用本项目建立团队共享的组件库、脚手架、监控面板与运维手册链接集合，配合标签体系按项目或业务域分类，新成员入职时可快速了解团队依赖的基础设施与常用工具链。

公开技术情报采集与竞品分析。安全研究员或产品经理可通过分类目录定期收录竞品官网、行业报告来源、技术社区讨论串等公开信息，借助链接可用性监控功能及时发现被屏蔽或迁移的站点，保持情报源的时效性与稳定性。

离线文档中心前置导航。在专有网络环境或无互联网接入的开发测试区域，OpenLink Navigator 可作为内部镜像站点的入口导航页，帮助工程师快速跳转至内部 GitLab、Jenkins、Nexus 等核心服务，避免记忆大量 IP 地址与端口号。

## 快速开始

以下命令演示了从代码仓库克隆、安装依赖到启动开发服务的完整流程。

```bash
git clone https://github.com/openlink-navigator/core.git openlink-navigator
cd openlink-navigator
pip install -r requirements.txt
python manage.py migrate
python manage.py loaddata resources/fixtures/initial_catalog.json
python manage.py runserver --host 0.0.0.0 --port 8080
```

## 贡献指南

1. 复刻主仓库至个人账号下，在本地新建功能分支并命名为 feature/你的功能简述，确保分支名称简洁且不包含特殊字符。

2. 针对新增资源分类或修改核心检索逻辑的变更，需在 tests 目录下补充对应的单元测试或集成测试用例，保证增量代码的行覆盖率不低于 85%。

3. 提交 Pull Request 之前，请运行 `make lint` 与 `make test` 进行代码风格检查与全量测试套件执行，确保本地所有检查项为通过状态。

4. 若需新增外链资源，请按照 resources/fixtures/example_entry.json 中的字段格式编写 JSON 条目，并通过管理接口或导入命令提交，禁止直接修改生产数据库。

5. 文档类变更（包括 README、API 示例、部署说明）需同步更新 docs 目录下对应的 Markdown 文件，并在 commit message 中标注 [docs] 前缀以便区分。

## 常见问题

**问：系统提示 “检测任务队列堆积” 应如何处理？**

答：该现象通常是由于 Redis 连接超时或 Celery Worker 进程异常退出所致。建议首先检查 Redis 服务状态与网络连通性，其次查看 logs/celery.log 中的错误堆栈。若为偶发性问题，可手动执行 `python manage.py flush_tasks` 清空当前任务队列并重启 Worker 进程。

**问：静态站点导出后的页面无法正常显示分类层级？**

答：请确保在执行导出命令时已通过 `--depth` 参数正确指定递归层级，且 resources 数据中所有 Category 记录的 parent 字段指向有效 ID。如果分类树中包含循环引用，导出程序会自动截断并记录警告日志，可在导出日志中查看具体异常分类名称并修正数据。

**问：如何将现有浏览器书签批量导入系统？**

答：本项目未直接提供浏览器书签解析器，但可使用 `scripts/import_bookmarks.py` 辅助脚本。该脚本支持 Netscape 格式的 HTML 书签导出文件，通过 `--file bookmarks.html --tags import` 命令将其转换为资源条目，转换规则为：文件夹名映射为分类，书签标题与 URL 分别映射为资源名称与地址。
