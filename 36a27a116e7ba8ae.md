# LinkVault

LinkVault 是一个面向技术社区与内容创作者的轻量级外链资源聚合与导航平台。该项目定位为可自部署的垂直领域链接收藏夹与分类索引系统，帮助技术团队、独立开发者、内容运营者高效管理、展示和共享高质量的互联网资源。LinkVault 不提供爬虫或采集功能，仅作为人工筛选与组织的外部链接目录，适用于构建技术文档站、工具导航页、个人书签墙或团队知识库的外部引用索引层。

## 功能概览

- **多级分类与标签体系**：支持为每条外链配置分类、子分类及自由标签，便于按技术栈、领域或用途快速筛选。分类结构采用树形存储，支持无限层级嵌套，满足从粗粒度到细粒度的组织需求。

- **链接状态健康检查**：内置定时或手动触发的链接可用性探测，自动标记失效或响应超时的链接，并生成异常报告。探测策略支持 HTTP 状态码校验、关键词匹配及响应时间阈值判定。

- **全文与元数据检索**：基于链接标题、描述、分类路径、标签及备注字段提供中文分词全文检索，同时支持按添加时间、点击量、状态等维度排序与过滤。

- **访问统计与点击追踪**：为每条外链记录点击次数、最近访问时间及来源 IP 归属地域概览，数据聚合后展示为趋势图表，辅助判断资源热度与用户关注方向。

- **API 优先的开放数据接口**：提供 RESTful API 用于链接的增删改查、分类管理及状态查询，所有数据输出支持 JSON 格式，便于嵌入第三方系统或作为数据源供给前端应用。

- **批量导入与导出**：支持通过 CSV 或 JSON 格式批量导入链接数据，字段映射可配置；同时支持按分类、标签或时间范围导出数据子集，便于备份或迁移。

- **用户权限与团队协作**：提供基于角色的访问控制（管理员、编辑者、访客），支持多用户协作维护链接库，操作日志完整记录，便于审计与回溯。

## 应用场景

- **技术团队内部文档站的外链索引**：开发团队在维护技术文档或 Wiki 时，可将分散在多个外部站点的参考文档、工具手册、API 规范通过 LinkVault 统一收录，并为文档中的引用提供稳定的短链或分类入口，减少重复搜集成本。

- **独立开发者构建个人工具导航**：独立开发者可将日常使用的云服务控制台、代码片段生成器、在线数据库设计工具、性能测试平台等资源按开发阶段（开发、测试、部署、监控）分类，形成个人化的高效工作面板。

- **社区或博客的友情链接与资源推荐页**：技术社区运营者或技术博主可使用 LinkVault 生成公开的资源推荐页面，分类展示合作伙伴、常用学习站点、开源项目镜像等，配合访问统计了解用户兴趣倾向。

- **技术培训与教学环境中的资源分发**：在培训机构或高校实验室中，讲师可将课程所需的实验平台、在线编译器、数据集的下载入口通过 LinkVault 集中管理，学员无需记忆大量 URL，通过分类即可快速定位。

## 快速开始

以下步骤指导您在本地开发环境或生产服务器上从源码部署 LinkVault。

```bash
# 克隆代码仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装项目依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化配置文件与数据库
cp .env.example .env
python manage.py migrate
python manage.py loaddata initial_categories.json

# 启动开发服务器
python manage.py runserver 0.0.0.0:8000
```

访问 `http://localhost:8000` 即可进入管理控制台，默认管理员账号为 `admin`，密码为 `admin123`（首次启动后请及时修改）。

## 安装要求

|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，建议使用 3.10 或 3.11 LTS 版本 |

| Supervisor | 4.2 及以上 | 进程守护工具，用于管理 Celery Worker 和 Beat 进程（生产环境推荐） |

## 项目结构

项目采用 Django 标准应用布局，核心模块拆分清晰，便于二次开发与维护。

```
linkvault/
├── manage.py                      # Django 项目管理入口
├── config/                        # 项目全局配置
│   ├── settings.py               # 基础配置（含数据库、缓存、中间件）
│   ├── settings_prod.py          # 生产环境覆盖配置
│   └── urls.py                   # 根路由映射
├── apps/
│   ├── links/                    # 链接管理核心应用
│   │   ├── models.py             # Link, Category, Tag, LinkStatus 模型定义
│   │   ├── admin.py              # Django Admin 后台定制
│   │   ├── services/             # 业务逻辑服务层
│   │   │   ├── health_check.py   # 链接健康检查实现（异步）
│   │   │   ├── importer.py       # CSV/JSON 导入解析器
│   │   │   └── stats.py          # 点击与访问统计聚合
│   │   └── api/                  # RESTful API 视图与序列化器
│   │       ├── views.py
│   │       └── serializers.py
│   ├── accounts/                 # 用户认证与权限管理
│   │   ├── models.py             # 扩展用户模型，集成角色字段
│   │   └── permissions.py        # 自定义权限类（基于角色）
│   └── common/                   # 通用工具与基础组件
│       ├── cache.py              # 缓存装饰器与 key 管理
│       ├── middleware.py         # 请求日志与限流中间件
│       └── validators.py         # URL 格式与安全校验
├── static/                       # 静态资源（CSS, JS, 图片）
│   ├── css/
│   └── js/
├── templates/                    # Django 模板（管理后台与公共页面）
│   ├── admin/                    # Admin 模板覆盖
│   └── public/                   # 前台分类展示页
├── scripts/                      # 运维与初始化脚本
│   ├── init_db.sh               # 数据库初始化与测试数据加载
│   └── celery_worker.sh         # Celery Worker 启动脚本
├── requirements.txt              # Python 依赖列表（生产）
├── requirements-dev.txt          # 开发额外依赖（测试、代码检查）
└── docker-compose.yml            # 开发环境容器编排（PostgreSQL + Redis + app）
```

## 常见问题

**Q：LinkVault 是否提供在线演示站点？如何快速体验完整功能？**

A：项目官网提供公共演示环境，地址为 `https://demo.linkvault.example`。您可以使用演示账号 `demo@linkvault` / `demo123` 登录，该账号拥有管理权限，可自由添加、编辑和删除链接，所有数据每 24 小时重置一次。同时，您也可以参考快速开始章节在本地 5 分钟内完成部署。

**Q：链接健康检查的工作原理是什么？是否会对目标站点造成压力？**

A：健康检查采用 Celery 周期性任务，默认每 6 小时执行一次。检查时发送 HEAD 请求（若目标不支持则降级为 GET 并设置超时 5 秒），仅验证 HTTP 状态码和响应头，不下载响应体。检查间隔和超时均可配置，单次检查并发数通过信号量控制，避免对目标站点产生异常流量。

**Q：如何将 LinkVault 的数据迁移到其他系统，或与其他工具集成？**

A：LinkVault 提供两种数据导出方式：通过管理界面的导出功能可生成 CSV 或 JSON 文件，包含所有链接与分类信息；通过 REST API 可编程获取全量或增量数据（支持分页和字段过滤）。此外，项目支持 Webhook 事件通知，可在链接新增、更新或状态变化时主动推送数据到指定端点，便于与 Slack、飞书或自建机器人集成。
