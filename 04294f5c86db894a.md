# Vanguard Navigator

Vanguard Navigator 是一个面向技术调研、数据采集与互联网资源聚合的开源外链索引与导航系统。项目定位于为开发者、数据分析师与运维人员提供一套轻量化、可自托管的优质互联网资源导航面板，帮助用户快速定位与访问分布在各类小众域名与新型顶级域（nTLD）上的视频流媒体、娱乐内容、体育资讯及网络工具站点。本项目不提供任何内容存储与分发服务，仅作为公开互联网资源的地址索引与结构化展示工具，严格遵循 robots 协议与相关法律法规。

## 功能概览

- **多级分类导航体系** 支持按媒体类型、内容主题与业务场景自定义分类层级，便于用户根据使用意图快速筛选目标站点。
- **站点存活状态检测** 集成主动式 HTTP/HTTPS 探活机制，周期性检测索引站点的可访问性，并以状态标记直观呈现给终端用户。
- **外链导入与批量管理** 提供基于 Markdown 列表与 CSV 格式的链接批量导入接口，支持外部链接的增删改查与标签管理。
- **响应式自适应界面** 基于 CSS Flexbox 与 Grid 构建，适配桌面端、平板与移动端浏览器，确保在不同屏幕尺寸下的良好阅读与操作体验。
- **站点元信息扩展字段** 支持为每个索引条目记录标题、描述、分类标签、添加时间与最后验证时间，便于用户进行二次筛选与排序。
- **纯静态生成模式** 支持构建时预渲染全量导航页面，输出为纯静态 HTML 文件，可直接部署于 Nginx、Apache 或对象存储服务，无需后端运行时依赖。
- **RESTful 管理 API** 提供标准化的 JSON API 接口，允许用户通过脚本或自动化工具远程管理索引数据，便于集成至现有运维或数据采集流水线。

## 应用场景

- **技术调研与竞品分析** 数据分析师或市场研究人员可通过 Vanguard Navigator 快速访问分散于非常规顶级域下的流媒体与娱乐站点，了解内容分发趋势与域名使用策略，为业务决策提供外部参考。
- **运维巡检与链路验证** 运维团队可将本项目部署为内部导航工具，集中管理各类测试环境、监控面板与第三方服务地址，利用内置的存活检测功能快速发现不可用节点。
- **个人知识库与书签管理** 开发者或技术爱好者可使用本项目搭建个人化的外链收藏系统，对高频访问的技术文档、工具站与社区资源进行结构化归档，替代浏览器自带书签的扁平化管理方式。
- **内容聚合与资讯采集** 编辑或内容运营人员可将项目用作资讯源地址池，定期导出索引列表供爬虫或 RSS 订阅工具使用，实现垂直领域信息的集中化采集与分发。

## 快速开始

以下操作基于 Ubuntu 22.04 LTS 与 Node.js 18.x 环境，其他 Linux 发行版或 macOS 系统可参考调整。

```bash
# 克隆代码仓库至本地
git clone https://github.com/vanguard-navigator/navigator.git
cd navigator

# 安装项目依赖（使用 npm）
npm install

# 执行构建流程，生成静态导航页面
npm run build

# 启动本地开发服务器，预览导航面板
npm run dev
```

完成上述步骤后，使用浏览器访问 http://localhost:3000 即可查看导航主界面。如需自定义索引数据，请编辑 `./data/links.json` 文件，随后重新执行构建命令。

## 常见问题

**问：项目是否提供在线演示站点？**

本项目不提供官方托管的在线演示实例，旨在鼓励用户自行部署以获得完整的数据控制权。用户可按照快速开始章节的指引在本地环境运行，或将其部署至自有服务器。如需测试前端界面效果，可使用项目附带的示例数据启动开发服务器进行预览。

**问：如何确保索引站点的内容合规性与安全性？**

本项目仅为公开互联网资源的地址索引工具，不抓取、缓存或代理任何第三方内容。用户自行添加的链接需遵守所在地法律法规。项目内置的存活探测仅验证 HTTP 状态码与响应时间，不解析页面正文。建议部署方定期审查索引列表，移除不可访问或违反规定的站点条目。

**问：能否自定义界面主题或品牌标识？**

可以。项目采用 CSS 变量管理主题颜色与字体，用户可编辑 `src/static/styles/variables.css` 文件中的色值与字号变量以调整视觉风格。此外，`src/templates/index.html` 中的页面标题、页脚信息与图标均可按需替换。对于深度定制需求，建议直接修改 UI 组件代码并重新构建。
