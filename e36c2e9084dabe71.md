# WebRes Bridge

WebRes Bridge 是一个面向技术调研人员、数据挖掘工程师与信息聚合爱好者的外链资源导航与元数据采集项目。项目定位为“技术资源的外链汇总站”，聚焦于对特定领域在线媒体资源、影视信息站点与流媒体入口的自动化可达性监测、基础元数据抽取与结构化输出。目标用户包括需要批量维护外链列表的运维人员、进行区域内容可访问性研究的数据分析师，以及构建轻量级影视信息聚合应用的原型开发者。

项目不提供具体视听内容，亦不涉及版权素材的存储与分发，仅对外链的 HTTP 响应状态、DNS 解析记录、SSL 证书有效期、页面标题与 meta 描述字段进行周期性采集，并以 JSON Lines 格式输出监测结果。通过标准化的外链元数据描述，本项目可有效降低重复性手工访问验证的成本，辅助快速识别链接状态变化，为上层应用提供稳定的基础数据输入。

## 功能概览

- 批量外链可达性探活：支持对预置外链列表并发执行 HTTP/HTTPS 请求，记录状态码、响应时间与重定向链，输出结构化日志。
- 基础元数据抽取：自动提取每个外链页面的文档标题、描述字段、字符集及 Content-Type，用于构建轻量级索引。
- DNS 与 TLS 信息记录：记录 A/AAAA 解析结果、证书颁发者与剩余有效期，便于追踪基础设施变更。
- 定时任务调度：内置基于 cron 表达式的周期性采集调度器，支持按小时、天或周粒度自动执行任务。
- 数据导出兼容性：采集结果支持导出为 CSV 与 JSON Lines 格式，兼容主流数据分析工具与 ETL 流程。
- 异常重试与容错：对超时、DNS 失败或 SSL 错误等异常情况自动进行指数退避重试，最多重试 3 次。
- 配置化外链管理：通过 YAML 配置文件管理外链列表，支持分类标签与启用/禁用状态控制，无需修改代码即可调整采集范围。
- 健康检查接口：提供简单的 HTTP 健康检查端点，便于接入 Prometheus 或其他监控系统进行可用性告警。

## 应用场景

1. 媒体信息站点可用性监控
   数据分析团队可利用本项目的周期性探活功能，对多个影视信息类网站进行持续可达性跟踪。当检测到站点返回 5xx 错误或连接超时时，系统自动记录异常时间戳与响应详情，帮助快速区分区域性网络波动与站点故障。

2. 外链资源清单的自动化校验
   内容聚合平台的内容运营人员经常维护大量外部链接列表。本项目可定期对清单中每一链接执行访问测试，输出状态报告，标记失效链接与重定向变更，从而协助运营人员及时更新页面上的跳转入口，减少用户点击后遇到 404 或空响应页面的情况。

3. 区域解析差异调研
   网络工程人员在研究不同 DNS 解析器对同一域名返回不同 IP 地址时，可利用本项目记录的多源 DNS 解析结果。通过对比不同采集节点（需配合多地域部署）的解析输出，分析 GeoDNS 或 CDN 调度策略的实际生效情况。

4. 证书有效期集中管理
   安全运维人员可将本项目作为辅助工具，集中监控多个域名的 SSL 证书剩余天数。当证书剩余有效期低于设定阈值时，采集日志会输出警告级别信息，便于提前安排证书更换操作，避免因证书过期导致服务不可用。

## 快速开始

以下步骤适用于 Linux/macOS 及 Windows WSL2 环境，需提前安装 Git 与 Go 1.21 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/webres-bridge/webres-bridge.git
cd webres-bridge

# 安装项目依赖（使用 Go Modules）
go mod download

# 使用默认配置文件运行采集任务（首次运行会输出示例结果到 stdout）
go run cmd/webres/main.go --config configs/example.yaml --run-once
```

若希望启动周期性调度模式，可执行以下命令：

```bash
# 启动调度器，按配置文件中 schedule.cron 表达式执行采集
go run cmd/webres/main.go --config configs/example.yaml --daemon
```

采集结果默认写入 `./output` 目录，按日期分文件夹存储，文件名为 `probe_YYYYMMDD_HHMMSS.jsonl`。

## 常见问题

问：采集过程中出现大量超时错误，应该如何处理？
答：首先检查本地网络环境，确认出站 443 与 80 端口是否通畅。若网络正常，可调整配置文件中的 `probe.timeout` 参数（单位秒），适当增加超时阈值。同时检查 `probe.max_redirects` 是否被目标站点大量重定向耗尽。项目支持通过 `--rate-limit` 参数控制每秒请求数，避免触发目标站点的流量限制。

问：如何只采集配置文件中的部分站点，而非全部？
答：在配置文件中为每个外链条目添加 `enabled: true/false` 字段，并在 `probe.filter.enabled_only` 设置为 `true`，采集器将仅处理标记为启用的站点。此外，可通过命令行参数 `--tags` 指定标签过滤，例如 `--tags video,chinese`，仅采集带有对应标签的条目。

问：采集结果中的页面标题和描述字段经常为空，是什么原因？
答：部分站点可能采用客户端 JavaScript 动态渲染页面内容，而本项目默认仅获取静态 HTML 并解析 meta 标签。对于纯 JS 渲染的页面，建议开启 `probe.render_js` 实验性选项（需额外依赖 Chromedp），但请注意这会显著增加采集耗时与资源占用。更推荐的方式是使用站点提供的结构化数据接口（如 JSON-LD）作为替代采集源。
