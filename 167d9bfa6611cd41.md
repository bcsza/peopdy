# FilmLink Navigator

FilmLink Navigator 是一个面向影视行业从业者、独立制片人及后期制作团队的技术资源导航站与外部工具链聚合平台。本项目不提供任何视频内容存储或分发服务，专注于收录、分类与维护影视制作全流程中所需的优质外部工具、素材平台与行业资讯入口。

项目目标用户为影视剪辑师、调色师、特效艺术家、制片统筹以及影视技术爱好者。通过结构化整理与持续更新的外部链接资源，帮助用户快速定位适合其工作流的工具平台，减少重复搜索成本，提升项目前期筹备与中期执行阶段的工具选型效率。

本项目采用纯静态前端架构，资源数据以结构化配置驱动，支持社区贡献与链接有效性众包检测。所有外链均以原始 URL 原样呈现，项目本身不代理、不缓存、不修改任何第三方平台内容。

## 功能概览

**外链分类导航** 按影视制作阶段（前期、拍摄、后期、宣发）对收录的链接进行一级分类，并支持多标签交叉筛选。

**链接状态检测** 每日自动对已收录链接进行可达性探测，在界面中标明异常状态，帮助用户避开已失效或迁移的资源入口。

**社区推荐提交** 注册用户可通过 Web 表单提交新的工具站或素材平台链接，经维护者审核后纳入导航目录。

**个人收藏夹同步** 支持用户将常用链接添加至个人收藏，并在不同设备间通过账户体系同步收藏列表。

**关键词快速检索** 针对链接标题、描述、标签及所属分类进行全文检索，支持拼音模糊匹配与首字母缩写查询。

**链接变更追踪** 对已收录链接的标题、描述或域名变更进行历史记录，用户可查看特定链接的变更日志。

## 应用场景

独立制片人在项目筹备期需要为团队筛选云协作平台与分镜素材库。通过 FilmLink Navigator 的后期制作分类，可快速对比多个外部工具的功能定位与访问入口，缩短决策路径。

剪辑师在远程协作中需要频繁访问代理剪辑工具与转码服务。本项目维护的链接集合覆盖主流云端剪辑与转码平台，用户无需记忆多个域名，直接通过导航页跳转。

影视技术爱好者希望了解行业常用的素材授权平台与特效素材站。通过浏览本项目的资源列表，可系统性接触多个垂直领域的知名平台，作为学习与实验的起点。

## 快速开始

以下指令适用于开发环境或本地自部署实例。生产环境部署请参考文档导航中的运维部署指南。

```bash
# 克隆项目仓库
git clone https://github.com/filmlink/navigator.git

# 进入项目目录并安装依赖
cd navigator
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行完成后，访问本地服务地址即可查看导航页面的开发版本。若需构建静态产出物，请使用 `npm run build` 命令。

## 贡献指南

项目欢迎外部贡献者参与链接收录维护、代码缺陷修复及文档改进。所有贡献需遵守以下流程：

第一，在 GitHub 仓库中查阅现有 Issues 列表，确认无人正在处理相同主题。若计划新增链接分类或调整标签体系，建议先开启一个讨论 Issue 说明意图。

第二，从主分支检出新的功能分支，命名风格为 `feature/链接分类名称` 或 `fix/问题简述`。所有代码变更需保持与现有 ESLint 规则一致，并补充对应的单元测试用例。

第三，对于链接增删改类贡献，需同步更新 `src/config/links.json` 文件，确保新增链接包含完整的标题、描述、分类标签及提交人备注。链接 URL 必须为原始地址，不得添加任何跟踪参数或中间跳转。

第四，提交 Pull Request 时请填写模板中的检查清单，包括是否通过本地构建、是否测试过链接可达性以及是否更新了相关文档。PR 需至少一名维护者审核通过后方可合并。

第五，文档类贡献（修正错别字、补充场景案例、完善 API 说明）可直接提交 PR，无需预先开启 Issue，但需在 PR 描述中标注为文档变更。

## 常见问题

Q: 为什么项目不提供任何视频或素材文件的直接下载？

A: FilmLink Navigator 定位为技术资源导航站，所有收录的内容均为第三方平台的入口链接。项目本身不存储、不转发、不代理任何受版权保护的媒体文件，仅作为工具发现与信息聚合使用。用户跳转至第三方平台后需遵守各平台自身的服务条款。

Q: 收录链接失效时应如何处理？

A: 用户可通过页面上的「报告失效」按钮提交反馈，系统会自动记录报告时间与报告人。维护者每周会核对高频失效报告并尝试联系原平台方确认状态，若确认永久迁移或关停，将从导航列表中移除该条目并更新相关分类统计。

Q: 能否申请将个人开发的工具站收录进导航？

A: 可以。请通过项目页面中的「提交链接」表单填写工具站名称、原始 URL、简短功能描述以及所属分类。提交后维护团队会在 5 个工作日内评估工具站的可用性、内容相关性与稳定性，评估通过后即纳入下一版更新。提交前请确认工具站本身不包含侵权或恶意代码内容。
