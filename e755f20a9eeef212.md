# NavSphere

NavSphere 是一个面向开发人员与技术研究者的高质量网络资源导航聚合平台。本项目不提供具体软件或库，而是围绕特定领域构建结构化、可检索的外部资源索引，旨在解决技术信息分散、优质入口难以追溯、垂直领域站点聚合度低的实际问题。NavSphere 定位于成为技术调研、竞品分析、数据采集与内容挖掘场景下的起始页与书签增强系统，服务于需要高频访问特定类别网络服务的中高阶用户。

## 功能概览

- **垂直领域分类索引**：按行业、服务类型与数据属性对收录站点进行多层标签划分，支持快速筛选与模糊匹配。

- **可配置资源抓取框架**：内置基于正则表达式与 XPath 的元数据抽取模板，用户可针对任意新站点自定义字段映射规则。

- **多源健康度监控看板**：对每个收录域名实施可用性探测、响应时间统计与 SSL 证书有效期追踪，并生成可视化趋势图。

- **批量外链检查与去重**：自动检测失效链接、重定向链过长与内容重复条目，输出标准化清理报告。

- **私有书签同步网关**：提供 RESTful API 与浏览器插件适配层，支持将本地书签批量导入并去重合并至平台分类体系。

- **自定义导航主题模板**：支持浅色/深色/高对比度三种界面主题，且允许用户通过 CSS 变量覆盖全局样式。

- **每日增量更新提醒**：基于 RSS 与站点变更嗅探，对已收录资源的内容更新频率进行摘要推送。

## 应用场景

- **技术调研与竞品分析**：产品经理或技术负责人可利用 NavSphere 快速定位同类工具站点，通过健康度看板横向对比服务稳定性，通过元数据抓取获取版本迭代记录。

- **数据采集管道前置配置**：数据工程师在编写爬虫或调用公开 API 前，可通过平台检索目标站点的 robots 规则、速率限制说明与示例响应结构，减少试错成本。

- **个人知识库外链整理**：研究人员可将散落在浏览器收藏夹、笔记软件中的数百个链接统一导入 NavSphere，借助批量去重与标签合并功能生成可维护的专题导航树。

- **运维监控辅助入口**：运维人员可将内部监控面板、日志系统、CI/CD 控制台等内部地址作为私有分类托管，结合健康度看板统一检查可达性，替代本地 hosts 或简陋的 bookmark 文件。

## 快速开始

以下命令适用于 Linux/macOS 及 Windows WSL 环境，请确保已安装 Git、Node.js 18+ 与 pnpm。

```bash
# 克隆主仓库
git clone https://github.com/navsphere/navsphere-core.git
cd navsphere-core

# 安装全部依赖（含开发工具链）
pnpm install --frozen-lockfile

# 启动本地开发服务器（默认监听 http://localhost:4173 ）
pnpm run dev
```

访问本地地址后，系统将自动加载内置示例分类与资源数据。如需使用完整外部数据源，请参考下一节安装要求配置环境变量文件。

## 贡献指南

我们欢迎外部贡献者参与索引优化、抓取模板补充与前端组件改进。请按以下步骤提交变更：

1. 在 GitHub Issues 中搜索现有议题，确认无人正在处理相同主题；若为新需求，请使用标准模板创建 Issue 并描述预期改动范围。

2. 从主分支 `main` 派生新功能分支，命名规则为 `feature/简要描述` 或 `fix/问题编号`，避免直接在主分支上提交。

3. 针对新增站点索引，需在 `/data/seed` 下补充对应的 JSON 条目，并至少在 `/tests/unit` 中提供一个元数据抽取用例。

4. 提交前运行 `pnpm run lint` 与 `pnpm run test:unit` 确保代码风格与单元测试通过，并更新相关文档中的资源列表或 API 示例。

5. 发起 Pull Request 至 `main` 分支，描述变更动机、测试结果以及是否影响现有功能。核心维护者将在 48 小时内进行 Review。

## 常见问题

**Q：平台是否自动更新外部站点的内容，还是仅存储链接？**

A：NavSphere 仅存储链接、标题、描述与用户自定义标签，不缓存或代理外部站点的完整页面内容。健康度探测只校验 HTTP 状态码、响应时间与 TLS 证书有效性，不会解析或存储返回的实体正文。

**Q：我能否完全离线使用 NavSphere 的所有功能？**

A：分类浏览、本地书签导入与标签管理功能在无外网环境下仍可正常工作。但健康度看板、RSS 更新提醒以及基于远程资源的元数据抓取需要外部网络访问能力，这部分功能将在离线模式下自动降级为只读状态。

**Q：如何迁移现有分类结构到另一套 NavSphere 实例？**

A：提供 `pnpm run export:data` 命令，可将全部分类树、资源条目与标签关系导出为单个 JSON 文件。目标实例使用 `pnpm run import:data --path=./export.json` 即可完成恢复，支持增量合并或全量覆盖两种策略。
