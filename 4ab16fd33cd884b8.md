# OpenLink Navigator

OpenLink Navigator 是一个面向数字内容研究者、媒体监测机构与互联网档案馆管理员的开源外链资源导航系统。该项目并非搜索引擎，亦非内容聚合平台，而是一个高度结构化的外部链接引用仓库，专注于对特定垂直领域内的公开视频、影视、动画与图像资源站点进行人工筛选与分类整理，并提供稳定的访问入口映射。

项目目标用户包括：需要长期追踪特定类型媒体站点可用性的运维人员、从事网络内容生态分析的学术研究者，以及希望建立自有轻量级媒体资源索引的开发者。OpenLink Navigator 不存储任何第三方内容，不提供代理或加速服务，仅作为公共互联网公开资源的引用登记系统，帮助用户在站点频繁迁移或域名变更的环境中维持对目标资源的持续访问能力。

## 功能概览

- **结构化外链登记**：每条链接均附带来源分类、首次收录日期与可用性状态标记，支持按标签与站点类型进行多维度筛选。
- **自动可用性探测**：系统内置轻量级 HTTP 状态检测模块，可定时对已登记链接进行可达性验证，并输出可用率统计报告。
- **多格式导入导出**：支持 CSV、JSON 与 Markdown 表格格式的链接清单批量导入与导出，方便与其他数据看板或监控系统集成。
- **标签化分类体系**：预设影视、动画、短片、图像素材、备用镜像等标签类别，用户亦可自定义扩展标签，满足不同研究视角下的分类需求。
- **链接变更追踪**：记录每条链接的域名、路径与协议变更历史，支持追溯特定站点在过去一年内的迁移轨迹。
- **访问策略备注**：允许为每条链接添加备注字段，说明是否需特殊 User-Agent、是否受地域访问限制、是否需启用 Cookie 等访问策略信息。
- **只读只写分离架构**：前端展示层与后端管理接口分离，支持多用户只读访问与管理员写入权限的独立配置，适用于团队协作场景。

## 应用场景

- **影视内容站点可用性监测**：媒体监测机构可使用 OpenLink Navigator 登记一批影视类资源站点，通过自动探测模块每日获取可用性状态，及时发现失效链接并触发告警，保障监测数据的连续性。
- **学术研究中的样本源管理**：从事网络内容生态与域名迁移行为研究的学者，可利用本项目的链接变更追踪功能，记录特定类型站点在不同时间节点的域名变化，为论文提供可回溯的数据支撑。
- **个人媒体索引库搭建**：内容爱好者可借助本项目的导入导出功能，将日常收集的影视动画相关站点统一整理为私有索引库，并通过标签分类实现快速检索，避免收藏夹散落与链接遗忘。
- **运维团队的外链巡检**：企业运维团队可将 OpenLink Navigator 部署为内部工具，定期巡检对外业务中引用的第三方资源链接，提前发现因对方域名到期或服务下架导致的外链失效问题。

## 快速开始

以下步骤指导您在本机快速启动 OpenLink Navigator 开发或测试实例。

```bash
# 克隆项目仓库
git clone https://github.com/openlink-navigator/navigator-core.git
cd navigator-core

# 安装依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 初始化本地 SQLite 数据库并导入示例链接数据
python scripts/init_db.py --sample-data

# 启动开发服务器（默认监听 127.0.0.1:8080）
python app.py
```

启动成功后，访问 `http://127.0.0.1:8080` 即可查看链接列表面板。管理员默认账号为 `admin`，初始密码在首次启动时输出于终端日志中。

| pyyaml | 6.0 及以上 | 用于解析配置文件中的分类规则与探测策略 |
| pytest | 7.0 及以上 | 仅开发与测试环境必需，用于执行单元测试与集成测试用例 |

## 项目结构

```
navigator-core/
├── app.py                     # 主入口，初始化 Flask 应用与路由注册
├── config/
│   ├── default.yaml           # 默认配置，含探测间隔、分页大小、日志级别
│   └── custom.yaml.example    # 自定义配置模板，用户可复制后覆盖默认值
├── models/
│   ├── link.py                # 链接实体模型，定义 URL、标签、状态、备注等字段
│   ├── history.py             # 变更历史模型，记录每次链接修改的时间与操作者
│   └── probe.py               # 探测结果模型，存储每次可用性检查的响应码与耗时
├── services/
│   ├── probe_worker.py        # 异步探测工作线程，周期性扫描待检链接队列
│   ├── import_export.py       # 批量导入导出服务，支持 CSV 与 JSON 格式转换
│   └── tag_manager.py         # 标签分类管理服务，提供标签增删改查与合并功能
├── routes/
│   ├── api_v1.py              # RESTful API 路由，供前端界面与外部系统调用
│   └── web_ui.py              # 管理界面路由，返回 HTML 模板渲染结果
├── static/                    # 前端静态资源，含 CSS 样式表与 JavaScript 交互脚本
├── templates/                 # Jinja2 模板文件，用于渲染链接列表与详情页面
├── scripts/
│   ├── init_db.py             # 数据库初始化脚本，可创建表结构并填充示例数据
│   └── migrate_db.py          # 数据库迁移脚本，用于版本升级时的结构变更
├── tests/                     # 单元测试与集成测试用例目录
├── requirements.txt           # Python 依赖清单
└── README.md                  # 项目说明文件（即本文档）
```

## 常见问题

**问：可用性探测是否会频繁触发目标站点的反爬机制？**

答：探测模块默认采用较低的请求频率（间隔不少于 10 分钟），且仅发送标准 GET 请求头，不携带自定义 User-Agent 以外的额外标识。探测请求不解析页面内容，仅获取 HTTP 状态码与响应时间，行为与普通浏览器访问无异。若目标站点明确通过 robots.txt 禁止探测，管理员可在配置文件中将该链接加入白名单排除探测。

**问：如何将现有收藏夹中的链接批量导入系统？**

答：支持 Chrome 书签导出 HTML 文件、Firefox 书签 JSON 导出以及通用 CSV 格式。请参考 `docs/user-guide.md` 中的导入章节，使用 `scripts/import_bookmarks.py` 工具进行转换。导入时系统会自动尝试识别标题与标签，但建议导入后人工复核分类准确性。

**问：项目是否提供 Docker 镜像以便快速部署？**

答：官方 Docker 镜像发布在 GitHub Container Registry 中，您可以使用 `docker pull ghcr.io/openlink-navigator/navigator-core:latest` 获取。镜像内已包含所有依赖与默认配置，启动时挂载外部卷用于持久化 SQLite 数据库文件即可运行。详细容器化部署步骤请参阅 `docs/deployment.md`。
