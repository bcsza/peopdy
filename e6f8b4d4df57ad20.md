# LinkVault

LinkVault 是一个面向技术内容创作者、开源项目维护者以及信息聚合平台运营者的轻量级外链资源管理与导航聚合系统。该项目旨在解决多源异构链接分散、手工维护成本高、资源失效难以追踪等痛点问题，通过结构化的分类管理与可视化的状态监控，帮助用户构建稳定、可扩展的外部资源索引体系。

LinkVault 不提供任何非法内容或流媒体服务，仅作为公开网络资源的导航入口与技术研究演示工具，适用于个人知识库构建、团队技术文档外链治理以及中小型开源项目的外部依赖管理场景。

## 功能概览

- 资源分类看板：支持按媒体类型、服务商、地域或自定义标签对链接进行多维度分组，并生成可视化统计概览。

- 链接存活检测：内置定时任务与手动触发机制，对收录的每一枚外部链接执行 HTTP 状态码校验，自动标记异常或失效节点。

- 访问量统计与排序：记录每个资源链接的被点击次数与最近访问时间，支持按热度或新增时间排序展示。

- 自定义元数据扩展：允许用户为每个链接添加备注、到期时间、维护人、备用地址等自定义字段，适配不同业务场景。

- 开放 JSON API：提供基于 RESTful 风格的查询接口，支持按分类、关键词、状态等条件筛选，便于第三方系统集成。

- 多用户权限管理：支持管理员、编辑者、访客三级角色控制，限制敏感资源的查看与修改权限。

## 应用场景

技术文档站点的外链治理：开源项目文档站中常包含大量第三方依赖链接、教程引用或社区资源，LinkVault 可统一收录并监控这些链接的有效性，避免文档中出现死链影响用户体验。

个人知识库的资源索引：使用 Obsidian、Notion 或本地 Markdown 管理技术笔记的用户，可将散落在各篇文章中的参考链接统一导入 LinkVault，生成一份可检索、可分类的中央资源目录。

团队内部服务导航：开发团队或运维小组可将内网监控面板、日志系统、代码仓库、CI/CD 流水线等内部工具的入口地址录入 LinkVault，通过权限控制限制访问范围，同时提供清晰的导航入口。

内容聚合站点的外链托管：面向特定领域（如编程教程、开源工具库、设计素材）的聚合类网站，可使用 LinkVault 作为后台链接管理引擎，前端通过 API 动态渲染分类导航页面，降低静态配置的维护成本。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，要求系统已安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 安装项目依赖
npm install

# 启动开发服务器（默认占用端口 3000）
npm run dev
```

启动成功后，访问 http://localhost:3000 即可进入 LinkVault 控制台。首次启动将自动初始化 SQLite 数据库并创建默认管理员账户（用户名: admin，密码: admin123），请在生产环境部署前及时修改默认密码。

## 项目结构

```
linkvault-core/
├── src/
│   ├── api/                     # RESTful API 路由定义与控制器
│   ├── core/                    # 核心业务逻辑：链接管理、检测引擎、统计聚合
│   ├── db/                      # 数据库迁移脚本与模型定义（SQLite）
│   ├── scheduler/               # 定时任务调度器（存活检测、统计清理）
│   ├── middleware/              # 鉴权、日志、跨域、限流等中间件
│   └── utils/                   # 工具函数：HTTP 请求封装、URL 解析、校验器
├── config/
│   ├── default.json             # 默认配置项（端口、检测间隔、分页大小）
│   └── production.json          # 生产环境覆盖配置（需自行创建）
├── public/                      # 静态资源目录（前端构建产物或默认样式）
├── docs/                        # 完整项目文档（含 API 手册、部署指南）
├── tests/                       # 单元测试与集成测试用例
├── scripts/                     # 运维辅助脚本（备份、迁移、数据导入导出）

```

## 常见问题

问：LinkVault 是否支持 MySQL 或 PostgreSQL 作为生产数据库？

答：当前版本默认使用 SQLite 以降低部署门槛，但核心数据访问层已使用 Knex.js 构建，支持多数据库适配。您可自行修改 config/production.json 中的 client 与 connection 配置切换至 MySQL 或 PostgreSQL，但需额外安装对应的数据库驱动包，并自行处理表结构迁移适配。官方将在后续版本中提供官方迁移指南。

问：如何迁移已有的链接数据（如 CSV 或 JSON 格式）？

答：项目提供了 scripts/import-data.js 脚本，支持读取指定目录下的 CSV 或 JSON 文件并批量写入数据库。您需要按照脚本注释中的字段映射格式准备数据文件，然后执行 npm run import -- --file=./data/links.json 即可完成导入。详细的字段映射说明请参阅 docs/operations/data-migration.md。

问：存活检测是否会消耗大量资源，是否可调节检测频率？

答：检测模块使用异步并发请求，默认每 6 小时执行一次全量检测，并发数限制为 10 个请求同时发出。您可以在 config/default.json 中调整 CHECK_INTERVAL_MS 和 MAX_CONCURRENT_CHECKS 两个参数来控制检测频率与并发度。对于超过 1000 条链接的大型实例，建议将检测间隔调整为 12 小时，并将并发数降至 5 以避免被目标服务商限流。
