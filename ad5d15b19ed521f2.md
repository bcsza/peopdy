# LinkForge

LinkForge 是一个面向技术团队与独立开发者的开源外链资源聚合与导航系统。项目定位为轻量级、可自托管的资源目录墙，帮助用户将散落在浏览器书签、团队文档、个人笔记中的高价值外部链接，统一整理为结构化、可检索、可共享的导航页面。LinkForge 本身不存储任何实际内容，仅提供链接元数据管理与前端渲染框架，适用于需要快速搭建内部技术资源库、团队知识导航或个人开发工具集的场景。

## 功能概览

- **多级分类与标签系统**：支持无限层级的目录结构，可为每条链接附加多个标签，实现灵活筛选与归类。

- **链接元数据自动抓取**：提交 URL 后自动拉取页面标题、描述与图标信息，减少手动录入成本。

- **全文检索与即时过滤**：基于标题、描述、标签、分类路径进行实时搜索，结果高亮显示。

- **访问状态健康检查**：定时对已收录链接进行可用性探测，标记失效或响应超时的资源，便于维护。

- **私有与公开可见性切换**：每条链接可独立设置访问权限，支持团队内部共享或完全对外开放。

- **导入导出与批量操作**：支持 JSON、CSV、HTML 书签格式的批量导入导出，以及分类移动、标签批量增减。

- **响应式前端主题**：提供暗色/亮色两套主题，适配桌面与移动设备，支持自定义 CSS 变量。

- **RESTful API 与 Webhook**：提供完整的增删改查 API 接口，并支持新增/更新时触发外部 Webhook，便于与其他系统集成。

## 应用场景

- **团队技术文档中心导航**：开发团队可将内部 Wiki、API 文档、设计稿、CI/CD 流水线、监控面板等链接统一收归至 LinkForge，按项目或服务分类，新成员入职时可快速获取所有关键入口。

- **个人开发工具箱聚合**：独立开发者或研究员可将常用在线工具（JSON 格式化器、正则测试、颜色转换、图标库、代码生成器）集中管理，配合检索功能，避免频繁切换书签或记忆域名。

- **开源社区资源站**：开源项目维护者可搭建社区驱动的外链墙，收录教程、视频、周边工具、插件生态等外部资源，访客可自由浏览与检索，降低社区信息获取门槛。

- **企业合规与审计辅助**：合规团队可将监管公告、政策文件、行业标准等外部参考链接统一归档，配合访问状态检查，及时发现变更或失效的官方页面。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户建议使用 WSL 或 Git Bash。

```bash
# 克隆仓库
git clone https://github.com/your-org/linkforge.git
cd linkforge

# 安装依赖（使用 npm）
npm install

# 运行开发服务器（默认端口 3000）
npm run dev
```

生产环境部署请参考 `docs/deployment.md` 中的 Docker 或 PM2 方案。

| Nginx / Caddy | 否 | 反向代理，用于生产环境的 SSL 终止与静态资源加速 |

## 常见问题

**问：LinkForge 是否支持在线预览或代理被收录的网页内容？**

答：不支持。LinkForge 仅存储链接的元数据（标题、描述、图标、URL 本身），不会抓取或代理页面正文。所有点击操作均直接跳转至原始外部网站。用户需自行遵守各外部站点的使用条款。

**问：如何从现有的浏览器书签或书签管理工具迁移数据？**

答：您可以从 Chrome、Firefox 或 Edge 导出 HTML 书签文件，然后在 LinkForge 的管理后台使用「导入」功能，选择 HTML 格式即可自动解析分类与链接。同时我们也支持 JSON 与 CSV 格式的导入导出，方便与其他工具对接。

**问：健康检查功能会对目标网站造成压力吗？**

答：健康检查默认采用 HEAD 请求，仅获取响应状态码，不下载页面内容。检查间隔默认为每 24 小时一次，且并发数受限，不会对目标服务器产生明显负载。您也可以自定义检查频率或完全关闭该功能。
