# LinkVault

LinkVault 是一个面向技术内容创作者、开源社区运营者和个人知识管理爱好者的外链资源聚合与导航工具。它并非简单的书签管理器，而是一个轻量级、可自托管的资源目录框架，旨在解决信息分散、链接失效、资源发现效率低下的问题。通过结构化的数据组织和清晰的分类视图，LinkVault 帮助用户将散落在各处的优质外链转化为有序、可检索、可共享的知识资产。

本项目定位为技术资源导航站点的元项目，提供标准化的资源清单模板、自动化链接状态检测脚本以及静态站点生成管线，适合用于搭建个人技术收藏夹、团队共享知识库或开源项目的外部依赖索引。

## 功能概览

- 资源分类目录树：支持无限层级的自定义分类，每条外链可归属多个标签，便于多维度筛选。
- 链接存活监控：定时对收录的 URL 发起 HEAD 请求，检测 404、超时、SSL 证书过期等异常状态，并在看板中高亮标记。
- 全文检索与过滤：基于标题、描述、标签和域名进行快速关键词匹配，支持按状态码、协议类型、更新时间排序。
- 批量导入与导出：支持从 CSV、JSON、Markdown 列表批量导入链接，导出为结构化数据文件，便于迁移或备份。
- 静态站点生成：内置模板引擎，可将资源数据渲染为纯静态 HTML 页面，适合部署到 GitHub Pages、Cloudflare Pages 或任意 Web 服务器。
- 开放 API 接口：提供 RESTful API 用于外部程序增删改查资源记录，便于集成到自动化工作流或浏览器插件。
- 用户自定义元字段：允许为每条链接附加自定义属性（如维护人、所属项目、阅读状态、重要性评分），满足个性化管理需求。
- 暗色主题与响应式布局：前端界面适配桌面与移动设备，内置浅色/深色两套主题，跟随系统偏好或手动切换。

## 应用场景

1. 个人技术博客的友情链接与推荐资源页管理。博主可以使用 LinkVault 维护一个动态更新的“好友链”和“技术阅读清单”，通过 API 或静态生成方式将最新列表嵌入博客侧边栏，避免手动更新 HTML 的繁琐。

2. 开源项目的外部依赖与文档镜像索引。大型项目通常引用大量第三方库、工具链和参考文档，LinkVault 可作为项目仓库的 companion 工具，集中记录所有外部 URL，并提供版本化的链接变更历史，帮助维护者追踪依赖来源的可用性。

3. 企业内部团队的知识库外链中枢。产品、研发、运维团队在日常协作中会积累大量 Confluence、Jira、Grafana 面板、日志平台等内部系统的链接，LinkVault 允许按团队或项目分类，并结合存活检测提前发现因权限变更或服务迁移导致的失效链接。

4. 技术社区或 newsletter 的每周资源精选集。社区运营者可以通过 LinkVault 的导入功能快速聚合成员推荐的优质内容，生成带分类标签和简短评注的资源列表，并一键导出为 Markdown 格式用于发布到论坛或邮件组。

5. 学术研究或论文写作的参考文献外链备份。研究人员可将引用到的在线资料、数据集仓库、工具主页统一存入 LinkVault，利用自定义元字段记录引用日期和访问状态，避免在论文修订过程中丢失重要参考来源。

## 快速开始

以下步骤帮助您在本地环境快速启动 LinkVault 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 pip 和 npm）
pip install -r requirements.txt
npm install --prefix frontend

# 初始化数据库并导入示例资源
python scripts/init_db.py
python scripts/import_demo_data.py

# 启动开发服务器（后端 + 前端）
python app.py --host 127.0.0.1 --port 8080
npm run dev --prefix frontend -- --port 3000
```

访问 `http://127.0.0.1:3000` 即可进入管理界面。默认管理员账号为 `admin`，密码为 `linkvault2026`，首次登录后请及时修改。

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。建议使用 Python 3.10 虚拟环境（venv 或 conda）隔离依赖，同时确保 Node.js 版本符合要求。

2. 创建新的功能分支，分支命名遵循 `feature/描述` 或 `fix/描述` 格式。提交代码前请运行 `pre-commit` 钩子执行代码格式检查（black、eslint、prettier）。

3. 编写或修改单元测试，确保新增功能的测试覆盖率不低于 80%。测试文件放置在 `tests/` 对应子目录下，使用 pytest 运行全部测试用例。

4. 更新文档。若新增了配置项、API 接口或修改了数据模型，请在 `docs/` 中同步更新相关章节，并在 PR 描述中说明文档变更点。

5. 提交 Pull Request 到主仓库的 `develop` 分支。PR 描述中需包含问题背景、解决方案、测试结果和影响范围。至少需要一位项目维护者审核通过方可合并。

## 常见问题

Q: 链接状态检测报告大量超时或 SSL 错误，该如何处理？

A: 请先检查服务器网络出口是否能够访问目标域名，部分境外站点可能需要配置代理。其次，检测器默认超时时间为 10 秒，对于响应较慢的服务可在 `config.yaml` 中调大 `checker.timeout` 值。若频繁出现 SSL 证书验证失败，可考虑设置 `checker.verify_ssl` 为 `false`（仅限内网或测试环境）。

Q: 如何从旧版本迁移数据，或者将数据库从 SQLite 迁移到 PostgreSQL？

A: 项目提供了 `scripts/migrate_db.py` 工具，支持在不同数据库 URI 之间迁移表结构和数据。具体用法为：设置环境变量 `SOURCE_DB_URL` 和 `TARGET_DB_URL`，然后执行迁移脚本。迁移前请务必完整备份原始数据文件。对于版本升级带来的 schema 变更，请查阅 `docs/upgrade.md` 中的逐版本操作步骤。

Q: 静态站点生成后，部分链接的图标或缩略图不显示，是什么原因？

A: 静态生成器默认使用 `screenshot` 服务获取网页预览图，该功能依赖外部 API（如 sshoter 或 mikas）。若未配置有效密钥或服务不可用，则会回退到纯文字样式。您可以在 `config.yaml` 的 `static_gen.preview_provider` 中更换提供方，或禁用预览图功能。另外，请确保生成时网络环境能够访问外网，否则无法抓取目标页面元数据。
