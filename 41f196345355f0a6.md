# CloudLink 开源技术导航站

CloudLink 是一个面向开发者、技术研究者与开源爱好者的轻量级技术资源导航与外部链接聚合平台。项目定位于提供高质量、高可用、可自托管的互联网技术入口，帮助用户在纷杂的信息环境中快速定位到有价值的工具、文档、社区与动态数据源。目标用户包括个人开发者、小型技术团队、教育机构及技术内容创作者，通过本项目可构建属于自身领域范围的垂直技术导航站，无需依赖商业服务即可实现稳定、透明、可扩展的资源路由与访问加速指向。

项目核心解决三个问题：第一，技术资源分散，缺少统一入口导致检索效率低下；第二，外部链接稳定性不可控，难以进行健康监控与快速替换；第三，团队内部共享技术栈依赖时缺少可维护的链接清单体系。CloudLink 采用纯静态生成与轻量后端结合方式，支持定时链接可用性探测、访问日志聚合、自定义分类与标签过滤，兼顾易用性与扩展性。

## 功能概览

**多源链接聚合管理**：支持手动添加、批量导入、分类标签管理，所有外链数据存储于结构化配置文件中，便于版本控制与协同维护。

**可用性健康检查**：内置异步任务调度器，定时对注册的外部链接发起 HEAD/GET 探测，记录响应状态、耗时与证书有效期，异常时触发告警通知。

**访问统计与热度排序**：基于访问日志记录链接点击频次，支持按周、月、全部维度统计，并生成热度排行与趋势折线图。

**自定义分类与标签体系**：允许用户自由创建一级分类及二级标签，同一链接可归属多个标签，支持模糊搜索与多条件过滤。

**主题切换与响应式布局**：内置浅色/深色两套主题，适配桌面、平板与移动设备，无需额外开发即可提供一致浏览体验。

**开放 API 接口**：提供 RESTful API 用于链接数据的新增、修改、删除及查询，便于与其他自动化工具（如 CI/CD、监控机器人）集成。

## 应用场景

个人技术博客扩展导航：开发者可在个人博客中嵌入 CloudLink 作为独立导航子站，集中整理常用开发工具、文档镜像、在线编译器、代码片段库等资源，提升博客实用性与用户粘性。

企业内部技术团队共享入口：小型研发团队利用 CloudLink 建立内部依赖服务地址簿，包含私有 Git 仓库、容器镜像源、项目管理工具、持续集成入口等，新成员入职时可快速熟悉基础技术设施。

技术教学与培训辅助平台：培训机构或高校实验室可搭建 CloudLink 实例，将课程所需实验环境、在线教材、API 参考文档、作业提交系统等链接统一收纳，降低学生查找成本，聚焦学习本身。

开源项目生态导航页：开源项目维护者可创建配套导航站，汇聚贡献者文档、社区论坛、CI 状态面板、代码覆盖率报告、版本发布日志等关键外部资源，完善项目治理与透明度。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/cloudlink-io/cloudlink.git

# 进入项目目录
cd cloudlink

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化配置并启动开发服务
cp .env.example .env
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 后端运行环境，核心逻辑依赖 Python 异步特性 |
| Django | 4.2 LTS | Web 框架，用于路由、ORM、管理后台及模板渲染 |
| Celery | 5.3 及以上 | 分布式任务队列，用于定时健康检查与异步日志处理 |
| Redis | 7.0 及以上 | 消息代理与缓存后端，配合 Celery 及会话存储 |
| PostgreSQL | 14 及以上 | 主数据库，存储链接元数据、分类、标签及访问日志 |
| Node.js | 18 LTS | 仅前端构建时需要，用于编译静态资源与主题样式 |
| Nginx | 1.24 及以上 | 生产环境推荐反向代理服务器，处理静态文件与负载均衡 |
| Supervisor | 4.2 及以上 | 进程守护工具，用于生产环境管理 Celery Worker 和 Beat |
| certbot | 2.0 及以上 | 可选，用于自动续期 Let's Encrypt TLS 证书 |

## 常见问题

Q: 健康检查任务对目标站点是否会产生较大请求压力？

A: 健康检查默认间隔为 30 分钟，每个目标链接每次仅发送一次轻量 HEAD 请求，超时设置为 5 秒。对于大规模部署（链接数超过 1000），建议调整 Celery 并发数并将检查间隔延长至 1 小时，同时配置 Redis 结果过期时间以控制内存占用。

Q: 如何从旧版导航系统迁移链接数据至 CloudLink？

A: 项目提供数据迁移脚本位于 backend/apps/links/management/commands/import_legacy.py，支持 CSV、JSON Lines 及通用书签 HTML 格式。执行 python manage.py import_legacy --input old_data.json --format json 即可导入，导入前建议先使用 --dry-run 参数预览转换结果。若自定义字段映射，可参考 docs/deploy/migration.md 编写映射配置文件。
